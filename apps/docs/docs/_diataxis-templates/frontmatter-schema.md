# Frontmatter schema

Every page carries this YAML frontmatter. Docusaurus consumes `id` / `title` /
`sidebar_*`; the three **governance** fields (`diataxis_type`, `package`,
`legacy_guard`) are what the CI gate greps and what lets site-level
Concepts/Tutorials be _surfaced_ per package (ADR-RXA-0004 Decision 4, "surface,
not store"; RFC-RXA-0001 §Conventions).

## Field table

| Field              | Required | Allowed values                                                        | Purpose / rule                                                                                                                                                                                                                                                              |
| ------------------ | -------- | --------------------------------------------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `diataxis_type`    | **yes**  | `tutorial` \| `how-to` \| `reference` \| `explanation`                | **Exactly one** per page. The load-bearing structural field (ADR-RXA-0001 Decision 1; ADR-RXA-0007 §2). CI asserts one-and-only-one.                                                                                                                                        |
| `package`          | **yes**  | `state` \| `template` \| `cdk` \| `isr` \| `eslint-plugin` \| `_site` | Owning package. Use `_site` for cross-package `tutorial`/`explanation` (they live site-level per ADR-RXA-0004 Decision 1). Attribution follows ADR-RXA-0003 (document a concept where the user meets it: transforms → State journey, render strategies → Template journey). |
| `legacy_guard`     | **yes**  | `false` **or** an audience string                                     | `false` = modern/first-class page. A non-empty string = a legacy-guarded page and is **both** the flag CI greps for AND the exact `audience=` passed to `<LegacyGuard>`. Use a canonical audience string (see below). (ADR-RXA-0006 Decision 3; ADR-RXA-0007 §2.)           |
| `id`               | yes      | kebab slug, unique                                                    | Docusaurus doc id; also the redirect key. Concept ids are `E1`–`E8` slugs; tutorial ids `T1`/`T2`/`T4` slugs.                                                                                                                                                               |
| `title`            | yes      | string                                                                | H1 / nav title. For **Reference**, the exact export/symbol name — never a paraphrase (fixes the wrong-H1 drift class, EVID-RXA-0004 item 12).                                                                                                                               |
| `sidebar_label`    | optional | string                                                                | Short nav label when `title` is long.                                                                                                                                                                                                                                       |
| `sidebar_position` | optional | int                                                                   | Order within a section. **Must be unique within a folder** (isr had pos-4/pos-5 collisions; CI flags dupes).                                                                                                                                                                |
| `tags`             | optional | list from initiative vocab                                            | `state, template, cdk, isr, eslint-plugin, structure, content, api-reference, guides, examples, migration, tooling, governance, scope`. Used to surface Concepts per package.                                                                                               |
| `concepts`         | optional | list of `E#` ids                                                      | The "surface, not store" hook: site-level Concepts to render in a per-package "Concepts" box. E.g. a state page sets `concepts: [E3]`.                                                                                                                                      |

## Hard rules the CI gate depends on

- `diataxis_type` is present and **single** on **every** page (CI check S1).
- If `legacy_guard` is a string, the page body **must** contain a `<LegacyGuard>`
  and a link to Concept **E2** (CI currency check C6).
- A `reference` page is **one type only** — it may not carry How-to steps or
  Explanation prose (enforced by the Reference template FORBIDDEN list + PURIFY).
- `sidebar_position` must be **unique within its folder** (CI check S4).
- `legacy_guard` value is exactly `false` or a **non-empty quoted string** (CI check S5).

## Canonical `legacy_guard` audience strings (reuse; do not invent per page)

- **Zone-config / zoneless / zone render-strategies (cdk):**
  `"Angular apps still running Zone.js (v≤20, or v21 with Zone opt-in) and brownfield migrations"`
- **The six `no-zone-*` eslint rules:**
  `"projects still running Zone.js — the rule is a no-op under zoneless (v21+)"`

## One filled example per Diataxis type

### Tutorial (site-level, cross-package — e.g. T1)

```yaml
---
id: T1-first-reactive-state-with-rxstate
title: 'Your first reactive state with RxState (alongside signals)'
diataxis_type: tutorial
package: _site # committed Tutorials T1/T2/T4 live under docs/tutorials/
legacy_guard: false
sidebar_label: 'First reactive state'
tags: [state, examples]
concepts: [E3]
---
```

### How-to (per package)

```yaml
---
id: connect-multiple-async-sources
title: 'Connect multiple async sources to state'
diataxis_type: how-to
package: state
legacy_guard: false
tags: [state, guides]
concepts: [E3, E6] # surfaces the assumed Concept boxes
---
```

### Reference (per package — regenerated from source)

```yaml
---
id: rx-state-api
title: 'RxState' # exact class name — never a paraphrase (drift item 12)
diataxis_type: reference
package: state
legacy_guard: false
tags: [state, api-reference]
---
```

### Explanation (site-level Concept — e.g. E2, a legacy-context concept)

```yaml
---
id: E2-zoneless-and-zonejs-change-detection
title: 'Zoneless & how Zone.js affected change detection'
diataxis_type: explanation
package: _site
legacy_guard: false # E2 itself is a current explanation OF legacy context, not legacy-guarded
tags: [cdk, eslint-plugin, content]
---
```

### Legacy-guarded Reference (cdk zone shelf — the string doubles as the component prop)

```yaml
---
id: zone-configurations
title: 'zoneConfig'
diataxis_type: reference
package: cdk
legacy_guard: 'Angular apps still running Zone.js (v≤20, or v21 with Zone opt-in) and brownfield migrations'
tags: [cdk, migration]
---
```
